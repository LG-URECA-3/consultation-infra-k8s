## 1. 전제 및 범위

이 문서는 `consultation-infra-k8s/terraform` 로 정의된 **EKS 기반 애플리케이션 인프라**를,
일반적인 프로덕션 소규모 트래픽 기준으로 24×7 운영한다고 가정했을 때의 **월 예상 비용(USD/KRW)** 을 정리한 것입니다.

- **환율 가정**: 1 USD = **1,470원**
- **리전**: `ap-northeast-2` (서울)
- **포함 범위**
  - VPC / 서브넷 / NAT
  - ALB, NLB
  - EKS 클러스터 (control plane) + system node group
  - Karpenter가 관리하는 애플리케이션 노드
  - CloudWatch Container Insights (메트릭 전용)
  - EKS 노드용 EBS (간단 가정)
- **포함하지 않은 범위 (참고)**  
  - MySQL / Redis / Kafka / Elasticsearch 등 **데이터 계층 EC2 비용** (별도 Terraform/구성으로 운영된다고 가정)
  - Route 53 / S3 / 외부 서비스 호출 등 부가 서비스 요금
  - 데이터 전송(Data Transfer) 비용 (리전 간/인터넷 Outbound 등)

아래 비용은 **대략적인 참고용**이며, 실제 과금은 트래픽/사용량/할인(리저브드, Savings Plans, 엔터프라이즈 계약 등)에 따라 달라질 수 있습니다.

## 2. 용량/운영 가정

### 2.1 EKS 노드 가정

- **system node group** (Managed Node Group)
  - 인스턴스 타입: `t4g.small` (2 vCPU, 2 GiB) — `variables.tf` 의 `system_node_instance_types` 기본값 기준
  - 개수: **2대** 상시 운영
  - 운영 시간: 730 시간/월 (24×7)
- **Karpenter app node**
  - 인스턴스 타입: `t4g.medium` (2 vCPU, 4 GiB)
  - 개수: 평균 **2대** 상시 운영 (피크 타임 스케일아웃 포함한 평균값으로 가정)
  - 운영 시간: 730 시간/월 (24×7)

> 실제 환경에서는 Karpenter 덕분에 피크 타임 외에는 노드 수가 더 줄어들 수 있으며,  
> 그 경우 여기서 계산한 값보다 비용이 낮아집니다.

### 2.2 네트워크/로드밸런서 가정

- NAT Gateway: 1개
  - 시간당 요금: **$0.062/h (~91원/h)** (Tokyo/인접 리전 기준 추정)
  - NAT 를 통해 나가는 데이터: **100 GB/월** (ECR pull, 패키지 다운로드, 외부 API 호촐 등 가벼운 사용량)
  - 데이터 처리 요금: **$0.062/GB (~91원/GB)** × 100 GB = **$6.20/월 (~9,100원/월)** (추정)
- ALB (Ingress, AWS LB Controller): 1개
  - 시간당 요금: **$0.028/h (~41원/h)** (서울 리전 근사치, Asia Pacific 요금대 참고)
  - LCU 요금: **$0.008/LCU-h (~12원/LCU-h)**, 1 LCU 사용 가정 (저~중간 트래픽)
- NLB (`db_nlb`): 1개
  - 시간당 요금: **$0.030/h (~44원/h)** (근사치)
  - NLCU 요금: **$0.006/NLCU-h (~9원/NLCU-h)**, 1 NLCU 사용 가정 (내부 트래픽 위주)

### 2.3 컨트롤 플레인 / 모니터링 가정

- EKS 클러스터 (standard support): **$0.10/h (~147원/h)**
- CloudWatch Container Insights (메트릭 전용)
  - `aws-cloudwatch-metrics` DaemonSet 사용
  - custom metrics 수: 수백 개 수준 가정 (pod/node 단위)
  - CloudWatch custom metrics 요금: **$5/월 (~7,400원/월)** 로 단순 근사 (실제는 metric 개수·주기에 비례)

### 2.4 EBS (노드 루트 볼륨) 가정

- 각 노드당 **gp3 20 GiB** 루트 볼륨
- 노드 수: 4대 (2×t4g.small + 2×t4g.medium)
- 총 용량: 80 GiB
- gp3 요금: **$0.10/GB-month (~147원/GB-month)** (근사치)

## 5. 민감도 분석 / 조정 포인트

### 5.1 Karpenter 노드 수/시간

- Karpenter 노드는 워크로드에 따라 자동으로 늘고 줄어듭니다.
- 위 가정(2대 × 24×7)은 **보수적인(다소 높은)** 편이고,  
  - 실제로 야간/비활성 시간대에 노드 수가 1대 이하로 줄어든다면,
  - app node 비용(약 $152/월 (~~223,400원/월))에서 **30~~50% 절감** 도 가능합니다.

### 5.2 NAT 트래픽

- NAT 를 통해 나가는 데이터(현재 100 GB/월 가정)가 크게 늘어나면,
  - 예: 1 TB/월 → 데이터 처리 요금이 $62/월 수준으로 증가.
- 이미지 캐싱, VPC Endpoint(S3/ECR) 활용 등으로 NAT 사용량을 줄이면 비용을 크게 절감할 수 있습니다.

### 5.3 CloudWatch Metrics

- Container Insights 메트릭 개수가 수천 개 이상으로 늘어나면,  
custom metrics 비용이 **수십 달러/월** 까지 올라갈 수 있습니다.
- 자주 보지 않는 메트릭/네임스페이스는 비활성화하거나 수집 주기/Retention 을 줄여 비용을 통제할 수 있습니다.

### 5.4 Reserved Instances / Savings Plans

- 현재 계산은 **온디맨드(On-Demand)** 기준입니다.
- 1~~3년 단위로 RI 또는 Savings Plans 를 적용하면,~~  
~~EC2/EKS 관련 비용에서 **30~~60% 수준의 절감** 도 가능합니다.

## 6. 데이터 계층(EC2) 비용 추정

이 절에서는 `consultation-infra-k8s` 에서 Terraform 으로 함께 생성하는 **MySQL / Redis / Kafka / Elasticsearch EC2 인스턴스**를  
EKS 클러스터와 동일한 기준(서울 리전, 온디맨드, 24×7 운영)으로 사용한다고 가정하고, **참고용 월 비용**을 계산합니다.

### 6.1 가정

- 공통
  - 리전: `ap-northeast-2` (서울)
  - 운영 시간: 730 시간/월 (24×7)
  - 인스턴스 패밀리: `t4g` (ARM, 온디맨드)
- 인스턴스 타입 및 대수 (Terraform 기본값 기준, `variables.tf`)
  - MySQL master: `t4g.medium` × **1대** (`db_instance_type = "t4g.medium"`)
  - Redis: `t4g.small` × **1대** (`data_instance_type = "t4g.small"`)
  - Kafka: `t4g.small` × **1대**
  - Elasticsearch: `t4g.medium` × **1대** (`es_instance_type = "t4g.medium"`)
  - (옵션) MySQL replica: `t4g.medium` × **1대** — `enable_mysql_replica = true` 인 경우에만 추가
- 데이터용 EBS 가정 (gp3, 0.10 $/GB-month (~147원/GB-month))
  - MySQL 데이터: **100 GiB**
  - Elasticsearch 데이터: **100 GiB**
  - Kafka 데이터: **100 GiB**
  - Redis 데이터: **20 GiB**
  - 합계: **320 GiB** → 별도 EBS 비용으로 계산

### 6.2 데이터 계층 EC2 + EBS 월 비용 (USD/KRW)

- **EC2 인스턴스 (compute)**  
  - `t4g.medium` 1대 (MySQL master)  
    - 단가: **$0.1042/h (~153원/h)**  
    - 월 비용: 0.1042 × 730 ≒ **$76.07 (~111,800원/월)**
  - `t4g.small` 1대 (Redis)  
    - 단가: **$0.0521/h (~77원/h)**  
    - 월 비용: 0.0521 × 730 ≒ **$38.03 (~55,500원/월)**
  - `t4g.small` 1대 (Kafka)  
    - 월 비용: **$38.03 (~55,500원/월)**
  - `t4g.medium` 1대 (Elasticsearch)  
    - 월 비용: **$76.07 (~111,800원/월)**
  - **소계 (데이터 계층 EC2 compute)**  
    - 2×t4g.medium + 2×t4g.small → **$228.20/월 (~335,500원/월)**  
    (76.07×2 + 38.03×2 ≒ 228.20)
- **데이터용 EBS (gp3 320 GiB)**  
  - 단가: **$0.10/GB-month (~147원/GB-month)**  
  - 용량: **320 GiB**  
  - 월 비용: 0.10 × 320 = **$32.00/월 (~47,000원/월)**
- **합계 (데이터 계층 EC2 + EBS)**  
  - **$228.20 + $32.00 ≒ $260.20/월 (~382,500원/월)**  
  - (MySQL replica 추가 시, `t4g.medium` 1대 → 약 **$76.07/월 (~111,800원/월)** 만큼 증가)

### 6.3 전체 인프라(애플리케이션 + 데이터 계층) 관점

- 애플리케이션/EKS 계층 합계 (섹션 4 기준): **$418.22/월 (~614,800원/월)**  
- 데이터 계층 EC2 + EBS 합계 (섹션 6.2): **$260.20/월 (~382,500원/월)**  

→ 두 항목을 단순 합산하면:

- **전체 인프라 온디맨드 기준 월 비용 ≒ $678.42/월 (~997,300원/월, 약 100만 원/월)**

> 데이터 계층은 실제 데이터량, TPS, 복제/샤딩 전략, 백업/스냅샷 정책에 따라 스펙과 비용 변동 폭이 큽니다.  
> 위 값은 Terraform 기본 인스턴스 타입과 간단한 디스크 가정(320 GiB)을 적용한 **대략적인 참고 수치**로 보시면 됩니다.

## 7. 예시: 3월 12일 ~ 3월 24일(13일간)만 운영 시 비용

위 모든 계산은 **한 달(30일, 24×7 상시 운영)** 기준 월 비용입니다.  
3월 12일 ~ 3월 24일까지 **13일 동안만** 동일 스펙으로 운영한다고 가정하면,  
월 비용에 단순 비례로 \frac{13}{30} 을 곱해서 대략적인 비용을 추정할 수 있습니다.

- **애플리케이션/EKS 계층**  
  - 월 기준: **$418.22/월 (~614,800원/월)**  
  - 13일 운영 시: 418.22 × \frac{13}{30} ≒ 181.2 → **약 $181.2 (≈ 266,400원)**
- **데이터 계층 EC2 + EBS**  
  - 월 기준: **$260.20/월 (~382,500원/월)**  
  - 13일 운영 시: 260.20 × \frac{13}{30} ≒ 112.8 → **약 $112.8 (≈ 165,700원)**
- **전체 인프라 (애플리케이션 + 데이터 계층)**  
  - 월 기준: **$678.42/월 (~997,300원/월)**  
  - 13일 운영 시: 678.42 × \frac{13}{30} ≒ 294.0 → **약 $294.0 (≈ 432,200원, 약 43만 원)**

> 실제 AWS 과금은 시간 단위/초 단위로 계산되며,  
> 위 값은 **‘월 비용 × (13일 ÷ 30일)’** 이라는 단순 비례 가정을 사용한 **근사치**입니다.
